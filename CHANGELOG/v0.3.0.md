# Release Notes — v0.3.0

> **IBM AI Services** · [Documentation](https://www.ibm.com/docs/en/aiservices/2026.06.0) · [Migration Guide](https://www.ibm.com/docs/en/aiservices/2026.06.0?topic=services-migration-guide-v020-v030)

---

## What's New

### 🗂️ AI Services UI

You can now deploy, manage, and monitor all your AI capabilities from a single web interface. The dedicated sections cover the full stack:

- **Digital Assistants** — deploy and monitor end-to-end AI assistant solutions; view real-time status, deployment history, and manage running instances
- **Services** — browse available AI capabilities in the built-in **Catalog** tab, then deploy them individually or as part of a larger solution
- **CLI integration** — all deployment operations are also available through the `ai-services` CLI
- **Secure by default** — access is protected with JWT token-based authentication (login, token refresh, revoke)
- **Persistent state** — application metadata and user records are stored in PostgreSQL, surviving restarts

---

### 💬 Smarter Conversational RAG

The chatbot now maintains conversation context across turns, so users can ask follow-up questions naturally without repeating themselves.

- **Context-aware answers** — pronouns and references from earlier in the conversation are resolved automatically
- **Multilingual** — conversations in **English, German, Italian, and French** are supported out of the box
- **Custom system prompts** — operators can tailor the assistant's persona and behaviour per deployment
- Prompt inputs are validated semantically before being sent to the model, reducing hallucination from malformed prompts

---

### 🔍 Similarity Search API

A new standalone service lets you run semantic search against your document corpus directly, independent of the chatbot.

- **Three search modes** — dense (k-NN), sparse (BM25), and hybrid
- **Optional reranking** — integrate Cohere reranker for higher-precision results
- **Simple API** — single endpoint: `POST /v1/similarity-search`
- Response headers include per-request timing so you can measure latency end-to-end

---

### 🔒 HTTPS Everywhere (For podman runtime)

All external communications are now encrypted by default.

- Services are fronted by a **Caddy reverse proxy** that handles TLS termination automatically
- Self-signed certificates are generated automatically for local/dev deployments
- Bring your own certificate for production environments
- HTTP/2 and HTTP/3 are supported

---

### 📄 Document Processing Improvements

The digitize and summarize services have been significantly upgraded.

**Digitization**
- DOCX files are now supported alongside PDF and plain text
- Table spans across multiple pages are now merged correctly
- Added support for multilingual table summarization to improve accuracy of table data in documents

**Summarization**
- Jobs now run **asynchronously** — submit a document and poll for results, no blocking calls
- Two strategies: *direct* (single-pass) and *chunked* (for long documents)
- Control summary length with the new `level` parameter (`brief`, `standard`, `detailed`)

---

### 💾 PostgreSQL Storage

The digitize and summarize services now use **PostgreSQL** as their primary storage backend, replacing the previous JSON file store.

- Concurrent document ingestion is safe — no more write contention
- Documents and jobs are queryable with full indexing
- ACID guarantees mean no partial state on failure

**Backup & Restore**
- Backup and restore capabilities across supported storage systems (PostgreSQL and OpenSearch) (not supported for summarize)
- Tools to support safe data migration and recovery  

---

## Improvements

### LLM Provider Support
- **PowerVS / WatsonX via LiteLLM** — the LiteLLM proxy now supports IBM PowerVS-hosted WatsonX models

### Security
- All services run as **non-root users** with arbitrary UID support (OpenShift-compatible)
- Passwords for OpenSearch and PostgreSQL are auto-generated on first deployment
- Secrets are stored as Kubernetes/OpenShift Secrets — never in plain config files
- vLLM API key authentication (optional, off by default)

---

## Breaking Changes

> ⚠️ **Upgrading from v0.2.0?** Follow the [Migration Guide](https://www.ibm.com/docs/en/aiservices/2026.06.0?topic=services-migration-guide-v020-v030) before deploying.

- **PostgreSQL is now required** for the digitize and summarize services. JSON file storage is no longer supported.
- **HTTPS is enabled by default** via the Caddy proxy. Existing plain-HTTP ingress rules need to be updated.
- New environment variables and CLI flags are required for catalog configuration.
